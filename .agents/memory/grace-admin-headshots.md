---
name: Grace admin headshots
description: Leadership/admin portrait photos supplied for the Grace site come with solid backgrounds; how to prep them before use.
---

- Admin/leadership headshots provided for the Grace High School site arrive as `.png` files that are RGBA but fully opaque (`identify` reports `alpha=False`) — the background is solid black, not actually transparent.
- **Prep step:** run the background-removal tool to get a transparent cutout, then `convert ... -trim +repage` to drop the empty margins.
- **Do NOT place a dark-hair cutout on a light card.** Background removal on dark hair against a dark/black backdrop yields a poor alpha matte: a jagged, stair-stepped hairline plus light "fringe" speckles. On a light frame this halo is glaring (the user called it "a blurry thing on top of his head"). Eroding/feathering the alpha alone does not remove the interior light fringe.
- **Reliable fix — composite onto a dark-green studio backdrop:** erode the alpha a bit (`-channel A -morphology Erode Disk:16 -blur 0x6 -level 20%,80% +channel`) to kill light fringe, build a vertical green gradient the same size (`magick -size WxH gradient:'#0C3A1D'-'#1C7340'`), then `magick bg.png eroded_person.png -composite out.png`. Dark hair on dark green blends seamlessly, dark jagged tufts disappear, and it's on-brand. Save as `*_studio.png` and point the `@assets/...` import there.
- **Why dark backdrop, not light:** dark-hair-on-light is high-contrast and unforgiving — every matte flaw shows. Dark-hair-on-dark-green is low-contrast and hides edge roughness.
- **How to apply:** check with `identify`; image ratio ~0.799 already ≈ the 4/5 card frame, so `objectFit: cover` + `objectPosition: center top` shows the whole portrait with natural green headroom. The card's own light gradient frame is hidden once the photo (with its green backdrop) fills it.
- **Always export the studio composite as 8-bit** (`-depth 8 -strip`, downscale to ~1000px wide). ImageMagick `gradient:` composites default to **16-bit** depth; a 16-bit PNG imported via the Vite `@assets` alias renders as a **broken image** (the card then shows the person's name twice — once as the `alt={name}` text of the broken img, once as the label below). It still works as a direct `public/` copy, which masks the bug — so the `cardtest.html` public-copy trick will NOT catch it. An 8.5MB 8-bit JPG loads fine, so it's the bit depth, not file size.
- **Verifying the card:** the "Our Leadership" section sits far below the 88vh hero, so app_preview screenshots can never reach it. The public-copy `cardtest.html` trick verifies layout but NOT the real `@assets` import path. To verify the actual import serves, `curl` the Vite fs URL: `curl -sI "http://localhost:80/@fs/home/runner/workspace/attached_assets/<file>.png"` should return `200 ... image/png`.
- **Refreshing an updated portrait — change the filename, don't overwrite in place.** When you improve a headshot but keep the same `@assets` filename, the user's browser keeps serving the cached image, so they report "no change." Save the new render under a versioned filename (e.g. `*_studio_v2.png`) and update the import — the new URL forces a fresh fetch. A user re-uploading your own prior output (recognizable by the exact gradient corner color `srgb(12,58,29)` = `#0C3A1D` and the earlier render's dimensions) is the tell that they're stuck on a stale cached version, not asking for a brand-new person.
- **Standardizing mismatched green backdrops across the leadership row:** supplied portraits arrive with *different* green backgrounds (one vignetted dark-corners, another with a bottom-left glow, etc.); side by side they look unprofessional. Fix = key out each green and re-composite every subject onto ONE identical brand-green studio gradient. Keying green to replace with green is very forgiving — any leftover edge green blends into the new green, so only subject-pixel holes matter.
  - **Key by HSL hue+saturation, not brightness:** split `-colorspace HSL` (after split, H=Red ch, S=Green ch, L=Blue ch), bg mask = hue in green band (`-fx "(u>=0.20 && u<=0.47)?1:0"` on H) AND `S -threshold ~14%`. This catches dark green corners (still green hue) while excluding warm skin (hue <0.1), cream/yellow knitwear (hue ~0.1–0.15, below 0.20), and neutral black clothing/hair (low S). Watch for green-leaf colors in floral prints punching holes — verify each result.
  - Clean the mask: `-negate -morphology Close Disk:3 -morphology Open Disk:2 -blur 0x1.2 -level 35%,65%`, then composite with the reliable two-image blend (`subject*M` + `newbg*(1-M)` via `-compose multiply` then `-compose plus`, both inputs `-alpha off`) to dodge the CopyOpacity graya bug.
  - Shared backdrop: `magick -size 1100x1375 -define gradient:center=550,470 -define gradient:radii=900,900 radial-gradient:'#1C7340'-'#0A3A1E'` — subtle studio glow behind the head, on-brand, identical for all three.
- **Recoloring a suit/tie: desaturate-to-neutral works, forcing-to-black fails.** Forcing a subject's navy suit -> pure black by hand crushed the fold detail into a flat blob (user: "suit, tie, shade all bad"); we have no img2img/inpaint that preserves a face, so a believable *black* recolor isn't achievable. But recoloring to a NEUTRAL grey/charcoal works great and keeps the fabric realistic:
  - Mask the garment by HSL hue+sat (blue band `H 0.50-0.78` AND `S > ~12%`) so skin (warm hue), white shirt (low S), and the green backdrop (H~0.33) are excluded and stay untouched. Clean with morphology Close/Open + slight blur.
  - Build a desaturated layer with `-modulate <bright>,0,100`: **saturation 0 keeps the original luminance = the fold/lapel shading** (this is why it looks real, not flat). Navy's luminance is dark (~52), so use `bright ~120` to lift it to a clear charcoal grey instead of near-black.
  - **Colorspace gotcha:** compositing the masked layer via a `-compose multiply` + `-compose plus` chain where the mask was made `-colorspace Gray` collapses the WHOLE output to grayscale (face/bg lose all color; pixels read `gray(N)` not `srgb(...)`). Use the 3-input masked composite instead: `magick base.png recolored.png mask.png -compose over -composite out.png` — keeps sRGB, recolors only where the mask is white. Verify by sampling `%[pixel:p{x,y}]` on face/corner/suit: face must stay `srgb` brown, corner `srgb` green.
